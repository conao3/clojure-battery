# clojure-battery モジュール整理メモ

このプロジェクトの目的は、Clojure で Python 標準ライブラリ風の "batteries included" 体験を提供することだと考えられる。
以下は、現在存在するモジュールと、それぞれが何をするとこのプロジェクトにとって価値があるかの整理。

## 方針

- Python 標準ライブラリとの対応を意識する
- API 名・返り値・例外・境界条件を Python に近づける
- 各モジュールに対応するテストを充実させる
- まずは「実用上よく使う部分」を優先する

## モジュール一覧と期待される役割

- [x] `abc`
  - 役割: 抽象基底クラス的な仕組みを提供する
  - このプロジェクトのためになること: プロトコル/型設計の共通基盤を作り、他モジュールの拡張性を上げる

- [x] `argparse`

- [x] `base64`

- [x] `binascii`

  - このプロジェクトのためになること: `base64` や `codecs` の土台になりやすい

- [x] `bisect`
  - 役割: ソート済みコレクションへの二分探索挿入

- [x] `builtins`

- [x] `cmd`

## 進め方メモ

- まず Python 標準ライブラリとの互換性の高い API を小さく揃える
- その後、例外メッセージ・境界条件・細かい挙動を Python に寄せる
- 各モジュールで「最小実用セット」を定義する
- テストで Python の期待値に寄せる
- README にモジュール対応表を作ると、プロジェクトの意図が伝わりやすくなる
